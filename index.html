Hvad bruger man semikolonner til i Java (og Processing)?

I Java (og også i Processing, da det bygger på Java) bruges semikolonner til at afslutte en kommando eller en linje med kode. Det fortæller, at en bestemt linje er slut, og den skal fortsætte med at analysere næste linje.

Hvad er datatyper, og hvilke datatyper kender du?

Datatyper angiver typen af data, en variabel eller en værdi kan indeholde. Nogle almindelige datatyper i Java inkluderer:

  • int: Heltal (f.eks. 5, -10)
  • double: Flydende kommatal (f.eks. 3.14, -0.5)
  • boolean: Sandt eller falsk (true eller false)
  • String: Tekst (f.eks. "Hello world")
  • char: Enkelt tegn (f.eks. 'a', '$')

Hvordan opretter man en variabel?

Du opretter en variabel ved at angive datatypen, variablens navn og eventuelt starte den med en værdi. For eksempel:

int alder = 25;

Hvad betyder "parametre" og "argumenter" i programmering, og hvornår anvender du dem i din kode?

Efter noget research har jeg fundet ud af at parametre og argumenter bruges i forbindelse med funktioner eller metoder. Parametre er variabler, der er deklareret i metode-definitionen og fungerer som pladsholdere for værdier, der sendes til metoden. Argumenter er de faktiske værdier, der overføres til metoden, når den kaldes.

Hvor har du brugt krølleparanteser (curly brackets) i din kode, og hvilken betydning har de?

Jeg bruger krølleparanteser til at definere blokke af kode. For eksempel bruges de til at definere kroppe af funktioner. Krølleparanteser markerer begyndelsen og slutningen af en kodeblok og hjælper med at organisere og afgrænse koden. Den gør det også meget nemmere at få et overblik over når man hurtigt læser sin kode.

Hvad betyder "variabel-scope," og betyder det noget i din kode?

Efter at have researchet på internet kan variable scope refererer til det område af koden, hvor en variabel er gyldig og kan bruges. I Java kan variabler have forskellige scopes som f.eks. global scope, klasse-scope eller metode-scope. Scopet påvirker, hvor længe en variabel eksisterer, og hvor den kan tilgås. Ja, variabel-scope betyder noget i min kode, da det hjælper med at undgå konflikter mellem variabler med samme navn og styrer, hvor variabler kan bruges.

Arrays (13/9/2023)

Arrays er en grundlæggende datatstruktur i programmering, der tillader dig at gemme flere værdier af samme type under ét navn. De fungerer som en liste, hvor du kan opbevare og håndtere flere elementer af data på en struktureret måde.

Dekleration af arrays

            int[]   listeHeltal;
            float[] listeKommatal:

Oprettelse af arrays

            listeHeltal = {1,2,3,9,0,4,5,8,7,6}; //værdier indsat
            listKommatal = new float[4]; //tom liste

Hvis du vil indsætte noget i et array gøres det ved skrive arrayet’s navn efterfulgt af firkantparanteser, der indeholder pladsnummeret. F.eks.:

        list[3] = 89;
        //dette indsætter tallet 89 på plads nummer 3
        list[8] = 11;
        //dette indsætter tallet 11 på plads nummer 8

For-loops er ofte brugt til at gennemgå alle elementer i en array og udføre handlinger på dem, f.eks. udskrivning, beregninger eller ændringer.

        int[] list={0,1,2,3,4};
        for( int i = 0 ; i < list.length ; i++){
           list[i] = list[i] + 1; 
        }

Resultatet af dette loop er, at hvert element i list bliver øget med 1. Efter løkken vil list indeholde [1, 2, 3, 4, 5].

"You have been given a String S. You need to find and print whether this string is a palindrome or not. If yes, print "YES" (without quotes), else print "NO" (without quotes)." (Skrevet i Java 8)

        import java.util.*;
        
        class TestClass {
            public static void main(String args[] ) throws Exception {
                Scanner in = new Scanner(System.in);
                String s = in.nextLine();
                String s2 ="";
                for(int i = s.length()-1 ; i >= 0 ; i--) {
                    char c = s.charAt(i);
                    s2 = s2 + c;
        
                }
        
                if(s.equals(s2)){
                    System.out.println("YES")
                }else{
                    System.out.println("NO");
                }
        
            }
        }

"You are required to enter a word that consists of x and y that denote the number of Zs and Os respectively. The input word is considered similar to word zoo if 2*x=y. Print Yes if the input word can be considered as the string zoo otherwise, print No." (Skrevet i Java 8)

        import java.util.*;
        
        class TestClass {
            public static void main(String args[] ) throws Exception {
                Scanner in = new Scanner(System.in);
                String s = in.nextLine();
                int zs = 0;
                int os = 0;
        
                for (int i = 0 ; i < s.length() ; i++){
                    if(s.charAt(i) =='z'){
                        zs++;
                    }else{
                        os++;
                    }
                }
                if(os == zs*2){
                    System.out.println("Yes");
                }else{
                    System.out.println("No");
                }
            }
        }

Referencebaserede og værdibaserede datatyper (29/9/2023)

I Processing Java er der to typer af datatyper: referencebaserede og værdibaserede. Referencebaserede datatyper, objekter, gemmes i heap-hukommelsen og indeholder en reference til deres faktiske data. Værdibaserede datatyper, primitiver, gemmes i stack-hukommelsen og indeholder selve dataen. Den afgørende forskel er, hvordan de håndterer data og hukommelse, hvilket påvirker, hvordan de manipuleres og overføres i koden.

I Java er arrays objekter, og når du opretter en reference til et objekt og tildeler det til en anden reference, deler de begge den samme reference til det samme objekt. Dette er, hvad der sker i tilfælde af arrays:

        int[] array1 = {1, 2, 3};
        int[] array2 = array1;
        array2[0] = 99;

Linje 2: array2 refererer nu til det samme array som array1

Linje 3: Dette vil ændre værdien i både array1 og array2

Hvad angår strengobjekter, så er strengobjekter i Java faktisk uforanderlige (immutable). Når du ændrer en streng, oprettes der i virkeligheden en ny streng, og den gamle streng forbliver uændret. Dette er grunden til, at når du ændrer en streng, påvirker det kun den reference, der er blevet ændret, og ikke andre referencer til den samme streng.

Arbejde med Primitiver: Opret to variabler af typen int og tildel dem samme værdi. Prøv at ændre værdien af den ene variabel og se, hvordan det påvirker den anden.

        int a = 1;
        int b = 1;
        a = b;
        println(a, b);

Arbejde med Primitiv Arrays: Opret et primitivt array, f.eks. int[], og tildel det til en anden reference. Prøv at ændre værdierne i det originale array og se, om det påvirker den anden reference.

        int[] a = {1,2,3};
        int[] b = {0,0,0};
        b = a;
        a[0] = 5;
        println(a, b);

Her vil der printes [0] 5, [1] 2, [2] 3

Funktioner (29/9/2023)

Eksempel på en funktion:

        int add2(int a, int b){
         int resultat = a + b;
        return resultat;
        }

Funtionens navn : anvendes til at “kalde” funktionen, når den skal anvendes

Parametre også kaldet metodeinput eller funktionsinput : kan bruges til at sende værdier til funktionen, kaldes “agrumenter”

Metodekrop eller funktionskrop : selve funktionens funktionalitet er skrevet inde i metodekroppen, som altid i Java markeres dette med krølle-paranteser

Returtype : I starten af funktionen ses hvad den returnerer i form af en bestemt datatype

Returværdi : Selve værdien der sendes tilbage fra funktionen kaldes returværdien
